Prework Study Guide

HTML

  • The head element contains information about the webpage.
  • The title element defines the title for a webpage.
  • The style element defines the CSS styles assocticated with a website.
  • The base element defines the base URL for a webpage.
  • The link element connects an external resource to the HTML document.
  • The meta element defines metadata such as the character set, description, keywords, authors, and viewpoint.
  • The script element embeds client-side scripts in the HTML document.
  • HTML can use six different levels of heading elements.
  • The body element represents the visible content shown to the user.
  • The paragraph element contains and displays a block of text.
  • The list element create list items
  • The HTML image elements embed images in documents

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector
  • Every HTML elemt is essentially a box
  • The img selector assigns property values to any image element in the markup.
  • The display property is assigned with the block value, which assigns each image to start on a new line.
  • CSS box model consists of margins outside the element, and the elements borders, padding, and content.
  • The text-align property specifies the horizontal alignment of text in an element
  • The font-size proprty sets the size of a font
  • A selector defines the element or attributes
  • Declaration contains the css property and value
  • DRY is a coding principle that encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintained
  • The display property is assigned with block value

Git

  • git status: checks what branch we are currently on
  • git branch branch name: creates new branch
  • git switch -c branch name: switches to the specified branch and updates the working directory
  • git checkout -b branch-name: creates a new branch and switches to it
  • git merge branch: combines the specified branch's history into the current branch
  • git log: lists verison history for the current branch
  • git log --follow file: lists version history for a file, beyond renames
  • git push: uploads all local branch commits to Github
  • git fetch: downloads all history from remote tracking branches
  • git pull: Updates your local working branch with all new commits from the corresponding remote brach on Github
  • git init: turn an existing directory into a git respository
  • git remote add orgin url: specifies the remote repository for your local repository
  • git clone: download a respository that already exists on Github
  • git commit -m decrpitive message: records file snapshots permanently in version history
  • git add file: snapshots the file in preparation for versioning
  • git reset commit: undoes all commits after commit, perserving changes locally
  • git reset --hard commit: discards all history and changes back to the specified commit
